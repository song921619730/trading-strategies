# 共识讨论与投票规则

## 为什么需要讨论？

单人的认知偏差很难自我纠正。三个人各自独立分析，然后互相看对方的分析，可以发现单人遗漏的问题。

## 讨论流程

### 第一轮：独立分析（并行）

三人同时接收相同的数据，各自独立输出分析结果。

```
Analyst    →  结构评分 + 入场/止损/止盈建议
RiskManager →  风险计算 + 硬过滤结果
President  →  趋势方向判断 + 大周期意见
```

### 第二轮：交叉评审（串行）

每人能看到其他两人的分析结果，回答：

> 看完其他两人的意见，你是否改变你的观点？
> 请给出最终决定： PASS(支持) / FAIL(反对)

```
Analyst 看到 RiskManager 说"RR<1:1，拒绝" → 可以坚持或降低评分
RiskManager 看到 President 说"趋势不支持" → 确定拒绝
President 看到 Analyst 评分 8 分但 RiskManager 拦截 → 可以坚持或调整
```

### 第三轮：最终投票

```
结果       → 动作
──────────────────────────────
3:0 全票   → 执行交易
2:1        → ❌ 不执行，记录少数派意见（为什么有人反对）
1:2        → ❌ 不执行
0:3        → ❌ 不执行
```

**全票通过的哲学**：一笔交易必须同时满足"结构好+风险可控+顺势"。三者缺一不可。

## 投票规则

- **每个人只有一票**：PASS（支持）或 FAIL（反对）
- **反对必须有理由**：写明违反哪条规则或什么问题
- **不允许弃权**：每个人必须给出明确决策
- **第二轮修改后，不再允许第三轮修改** — 必须在第二轮确定最终立场

## 讨论产出格式

每次共识讨论必须写入 `logs/consensus/` 目录，内容包含：

```yaml
session: "2026-05-07_1430"
symbol: "XAUUSD"
direction: "BUY"
round1:
  analyst:
    structural_score: 7
    verdict: "PASS"
  risk_manager:
    risk_pct: 2.3
    verdict: "PASS"
  president:
    d1_trend: "BULLISH"
    verdict: "SUPPORT"
round2:
  analyst: "看完 RM 的风险计算，确认结构支撑，维持 PASS"
  risk_manager: "看完 President 的趋势判断，确认趋势支持，维持 PASS"
  president: "看完 Analyst 的结构评分 7，结构合理，维持 SUPPORT"
final_vote:
  result: "3:0 UNANIMOUS"
  action: "EXECUTE"
  trade_ticket: "1758007434"
```

## 特殊规则

### 单方强势反对

如果某人的反对理由是基于量化规则而非主观判断，其反对视为"绝对反对"：
- RiskManager 说"RR<1:1" → 绝对反对，其他两人不用再劝
- RiskManager 说"SL设置不合理" → 绝对反对
- President 说"D1方向 CONFLICT" → 绝对反对
- Analyst 说"结构评分<4" → 绝对反对

### 需要重新讨论的情况

如果三人中有两人要求，可以就具体问题展开第二轮详细讨论（限时 5 轮来回），讨论的问题必须是：

1. "回调是否真的健康？" — 技术性讨论
2. "近端支撑/阻力强度如何？" — 技术性讨论
3. "手数是否可以调整以符合风险？" — 风控讨论

### 禁止讨论的话题

- 不允许说服某人放松规则
- 不允许"我觉得这次可以例外"
- 硬规则（RR≥1:1, 风险≤5%, SL证伪位）不可通过讨论绕过
