# data-mt5 — Researcher: Data Loading & Preparation

## Role

You are the **Researcher** profile. Your first job in every research round is to
**load and validate** the OHLCV parquet data, **compute technical indicators**,
and hand off a structured summary to the **Analyst** profile so they can formulate
and test hypotheses.

You do **not** analyse or draw conclusions — only prepare data and report what
is available.

## 1. Data Directory Layout

All parquet files live under the `data/` directory, organised by timeframe:

```
futures-intraday/
├── data/
│   ├── H1/
│   │   ├── XAUUSDm.parquet
│   │   ├── EURUSDm.parquet
│   │   ├── USDJPYm.parquet
│   │   └── ...            (14 symbols total)
│   ├── M30/
│   │   ├── XAUUSDm.parquet
│   │   ├── EURUSDm.parquet
│   │   ├── USDJPYm.parquet
│   │   └── ...
│   └── metadata.json       (generated by fetch_store_data.py)
├── scripts/
│   ├── data_loader.py
│   ├── grid_engine.py
│   └── fetch_store_data.py
└── skills/
    └── data-mt5.md          ← you are here
```

**Paths are relative to the `scripts/` directory** (the working directory for all
Python invocations). From `scripts/`, the data directories are:

| Timeframe | Relative Path                  |
|-----------|--------------------------------|
| H1        | `../data/H1/{symbol}.parquet`  |
| M30       | `../data/M30/{symbol}.parquet` |

Each parquet file contains a `pandas.DataFrame` with:

| Column       | Type      | Description                                  |
|--------------|-----------|----------------------------------------------|
| `time`       | DatetimeIndex (UTC) | Candle open time                      |
| `open`       | float64   | Open price                                   |
| `high`       | float64   | High price                                   |
| `low`        | float64   | Low price                                    |
| `close`      | float64   | Close price                                  |
| `tick_volume`| int64     | Tick count in the bar                        |
| `spread`     | int64     | Spread in points                             |
| `real_volume`| int64     | Real volume (if available from broker)       |

> **Important:** The `time` column is always the **UTC** candle open time. The
> date range is ~2021-01-01 to present (updated by `fetch_store_data.py`).

## 2. Available Scripts

### `../scripts/data_loader.py`

The primary data interface. Public functions:

| Function                 | Description                                                       |
|--------------------------|-------------------------------------------------------------------|
| `list_available_symbols(timeframe=None)` | Returns `List[str]` of symbols with parquet files on disk. Pass `'H1'` or `'M30'` to scope, or `None` for the union of both. |
| `load_data(timeframe='H1', symbols=None)` | Returns `Dict[str, pd.DataFrame]` mapping symbol → OHLCV DataFrame. If `symbols` is `None`, loads all available. |
| `compute_indicators(df)` | Adds technical indicator columns (ATR, RSI, MAs, Bollinger, session labels, consecutive counts) to a single DataFrame. Returns a copy. |
| `resample_to_daily(df)`  | Resamples intraday data to daily bars (OHLC + volume aggregation). |

**Import pattern:**
```python
from data_loader import load_data, compute_indicators, list_available_symbols, resample_to_daily
```

### `../scripts/grid_engine.py`

Hypothesis testing engine used **after** data is prepared (by the Analyst).
You will typically **not** call this directly, but you should be aware of it:

| Function    | Description                                                    |
|-------------|----------------------------------------------------------------|
| `run_grid(config)` | Evaluates an entry condition over loaded data, measures forward returns, returns aggregate stats. |

Import:
```python
from grid_engine import run_grid
```

### `../scripts/fetch_store_data.py`

**First-time data acquisition script.** Downloads 5 years of H1 + M30 data
from MetaTrader5 for all 14 symbols and saves as parquet.

**Do not run this if data already exists.** Only invoke when `data/H1/` or
`data/M30/` is empty (or has fewer than 2 symbols).

    print(f"{'='*72}\n")

print_data_summary(h1_data, "H1")

## 4. First-Time Data Fetch

### When to run

Check if data exists first. If `list_available_symbols()` returns fewer than
2 symbols (or if the `data/H1/` directory doesn't exist or is empty), you need
to fetch data.

### How to run

```bash
cd /mnt/f/AIcoding_space/Hermes/strategies/futures/research/kanban/futures-intraday/scripts
python3 fetch_store_data.py
```

**What happens behind the scenes:**

1. The script detects it is running under WSL Python (not Windows Python).
2. It **re-invokes itself** via `subprocess.check_call()` using the Windows
   Python interpreter at:
   `/mnt/c/Users/gj/AppData/Local/Programs/Python/Python312/python.exe`
3. Windows Python has access to the `MetaTrader5` library (which only works on
   Windows).
4. Data is fetched from MT5 for **all 14 symbols** for both **H1** and **M30**
   timeframes, from **2021-01-01** to present.
5. Parquet files are written to `../data/H1/{symbol}.parquet` and
   `../data/M30/{symbol}.parquet`.
6. A `metadata.json` file is written to `../data/metadata.json`.

**Do not try to run the script directly with Windows Python yourself** — the
script handles the re-invocation automatically. Just run `python3 fetch_store_data.py`.

### What to do after fetch completes

Wait for the script to finish (it prints a summary table), then re-run the
data loading steps (Section 3) to verify everything was saved correctly.

> **Note:** The fetch takes several minutes (14 symbols × 2 timeframes × ~5 years
> of data). Be patient. If it fails, check that MetaTrader5 terminal is running
> and logged in on the Windows side.

## 5. Computing Indicators

After loading raw OHLCV data, enrich it with technical indicators before handing
off to the Analyst.

### Basic usage

```python
from data_loader import compute_indicators

# Compute indicators for one symbol's DataFrame
df_raw = h1_data["XAUUSDm"]
df_enriched = compute_indicators(df_raw)

print(df_enriched.columns.tolist())
# ['open', 'high', 'low', 'close', 'tick_volume', 'spread', 'real_volume',
#  'atr14', 'rsi14', 'ma20', 'ma50', 'ma200', 'bb_upper', 'bb_lower',
#  'pct_chg', 'gap_pct', 'hour', 'dayofweek', 'session',
#  'consecutive_bull_count', 'consecutive_bear_count']
```

### Apply to all loaded symbols

```python
def enrich_all(data: dict) -> dict:
    """Apply compute_indicators to every DataFrame in the dict."""
    enriched = {}
    for sym, df in data.items():
        try:
            enriched[sym] = compute_indicators(df)
        except Exception as e:
            print(f"WARNING: compute_indicators failed for {sym}: {e}")
            enriched[sym] = df  # passthrough raw
    return enriched

h1_enriched = enrich_all(h1_data)
m30_enriched = enrich_all(m30_data)
```

### Indicator columns reference

| Column                   | Description                                                    |
|--------------------------|----------------------------------------------------------------|
| `atr14`                  | Average True Range (14-period, SMA of TR)                      |
| `rsi14`                  | Relative Strength Index (14-period, Wilder's smoothing)        |
| `ma20` / `ma50` / `ma200`| Simple moving averages of close                                |
| `bb_upper` / `bb_lower`  | Bollinger Bands (20,2) upper and lower envelopes               |
| `pct_chg`                | Close-to-close % change                                        |
| `gap_pct`                | (open - prev_close) / prev_close %                             |
| `session`                | Trading session: `'asia'` (00-07 UTC), `'europe'` (08-15 UTC), `'us'` (16-23 UTC) |
| `dayofweek`              | 0=Monday … 6=Sunday                                            |
| `hour`                   | Hour of day (0-23, UTC)                                        |
| `consecutive_bull_count` | Consecutive bullish candles (close > open)                     |
| `consecutive_bear_count` | Consecutive bearish candles (close < open)                     |

> **Note:** The first 200 rows of each enriched DataFrame will contain `NaN`
> values where rolling windows (e.g. MA200, ATR14) are not yet fully populated.
> This is expected. The Analyst can choose to drop or handle these.

## 6. Complete Workflow Script

Below is a complete, copy-paste-ready script you can run from the `scripts/`
directory. It performs all the steps above and prints the summary table.

```python
#!/usr/bin/env python3
"""Researcher: load data, compute indicators, print summary."""

import sys
from pathlib import Path
from data_loader import list_available_symbols, load_data, compute_indicators

# ------------------------------------------------------------------
# 1. Check data availability — fetch if empty
# ------------------------------------------------------------------
h1_syms = list_available_symbols(timeframe="H1")
m30_syms = list_available_symbols(timeframe="M30")

print(f"H1 symbols: {len(h1_syms)} — {h1_syms}")
print(f"M30 symbols: {len(m30_syms)} — {m30_syms}")

if len(h1_syms) < 2 and len(m30_syms) < 2:
    print("Data directory appears empty. Running first-time fetch ...")
    import subprocess
    ret = subprocess.run(
        [sys.executable, "fetch_store_data.py"],
        cwd=Path(__file__).parent,
    )
    if ret.returncode != 0:
        print("ERROR: fetch_store_data.py failed. Cannot proceed.")
        sys.exit(1)
    print("Fetch complete. Proceeding to load data ...\n")

# ------------------------------------------------------------------
# 2. Load raw data for both timeframes
# ------------------------------------------------------------------
print("Loading H1 data ...")
h1_raw = load_data(timeframe="H1")
print(f"  → {len(h1_raw)} symbols loaded")

print("Loading M30 data ...")
m30_raw = load_data(timeframe="M30")
print(f"  → {len(m30_raw)} symbols loaded")

# ------------------------------------------------------------------
# 3. Compute indicators
# ------------------------------------------------------------------
print("Computing indicators (H1) ...")
h1_data = {sym: compute_indicators(df) for sym, df in h1_raw.items()}
print("Computing indicators (M30) ...")
m30_data = {sym: compute_indicators(df) for sym, df in m30_raw.items()}

# ------------------------------------------------------------------
# 4. Print structured summary table
# ------------------------------------------------------------------
def print_summary(data: dict, tf: str):
    """Print a formatted data summary table for the Analyst."""
    print(f"\n{'='*78}")
    print(f"  RESEARCHER DATA SUMMARY — {tf}")
    print(f"{'='*78}")
    print(f"  {'Symbol':<12} {'Rows':>10} {'From':<20} {'To':<20} {'Sessions':>8}")
    print(f"  {'-'*70}")
    total_rows = 0
    for sym in sorted(data.keys()):
        df = data[sym]
        rows = len(df)
        total_rows += rows
        start = df.index[0].strftime('%Y-%m-%d %H:%M')
        end = df.index[-1].strftime('%Y-%m-%d %H:%M')
        # Count unique trading sessions present
        sessions = df['session'].nunique() if 'session' in df.columns else '?'
        print(f"  {sym:<12} {rows:>10} {start:<20} {end:<20} {sessions:>8}")
    print(f"  {'-'*70}")
    print(f"  {'TOTAL':<12} {total_rows:>10}  ({len(data)} symbols, {tf})")
    print(f"{'='*78}\n")

print_summary(h1_data, "H1")
print_summary(m30_data, "M30")

# ------------------------------------------------------------------
# 5. Report readiness for Analyst
# ------------------------------------------------------------------
print(f"Ready for Analyst. H1: {len(h1_data)} symbols, M30: {len(m30_data)} symbols.")
print("Enriched columns available: open, high, low, close, tick_volume, spread, "
      "real_volume, atr14, rsi14, ma20, ma50, ma200, bb_upper, bb_lower, "
      "pct_chg, gap_pct, session, dayofweek, hour, consecutive_bull_count, "
      "consecutive_bear_count")
```

Save this as `scripts/researcher_load.py` if you want to reuse it across rounds,
or run the commands interactively in a Python REPL.

## 8. Common Issues & Troubleshooting

| Symptom                                      | Likely Cause                                   | Fix                                                                 |
|----------------------------------------------|------------------------------------------------|----------------------------------------------------------------------|
| `list_available_symbols()` returns `[]`      | Data directory missing or empty                | Run `python3 fetch_store_data.py` (see Section 4)                    |
| `load_data()` returns `{}`                   | No parquet files found for the given timeframe | Check `../data/H1/` and `../data/M30/` exist                         |
| `compute_indicators()` raises KeyError       | DataFrame missing expected columns             | Ensure parquet files were created by `fetch_store_data.py` (standard schema) |
| `MetaTrader5` import error                   | Running under WSL Python, not Windows Python   | The script auto-reinvokes; if it fails, run from Windows terminal directly |
| Fetch takes > 10 minutes                     | MT5 rate limits or network latency             | Normal — 28 parquet files (14 symbols × 2 timeframes) take time. Let it finish. |
| Parquet file has fewer rows than expected    | Symbol was added recently to MT5               | Acceptable; the symbol will have a shorter history                   |
