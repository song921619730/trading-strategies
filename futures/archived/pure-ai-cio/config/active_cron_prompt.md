## 9. 组合风险控制

### 风险计算 (必须严格执行)
**开仓前必须计算实际风险金额，不得使用"默认手数"！**

1. **从 JSON 读取品种数据**：
   - `trade_contract_size`: 1 手合约规模 (如白银=5000 盎司，黄金=100 盎司)
   - `trade_tick_value`: 1 点变动价值 (如白银=$1/点/手)
   - `point`: 最小报价单位

2. **计算实际风险**：
   ```
   SL_Distance = abs(Entry_Price - SL_Price)
   Risk_Per_Lot = SL_Distance / point * trade_tick_value
   Actual_Risk = Risk_Per_Lot * Lots
   Risk_Pct = Actual_Risk / Equity * 100%
   ```

3. **动态调整手数**：
   - 目标风险：净值 1%~3% (根据信心度调整)
   - 最大风险：**净值 5%** (硬上限)
   - 计算公式：`Target_Lots = (Equity * Target_Risk_Pct) / Risk_Per_Lot`

4. **示例 (白银 XAGUSDm)**：
   - Entry: 80.216, SL: 78.799 → SL_Distance = 1.417
   - Contract: 5000 oz, Tick: $1, Point: 0.001
   - Risk_Per_Lot = 1.417 / 0.001 * $1 = $1,417 (1 手风险！)
   - 若目标风险 $20 (1%) → Lots = $20 / $1,417 = **0.014 手**
   - ⚠️ **AI 错误示例**: 开 0.10 手 = $141.7 风险 (7.3%) → 严重违规！

### 硬性限制
- 单笔最大风险 ≤ 净值 5%
- 强相关品种合并计算风险
- 总组合风险 ≤ 净值 20%
- 若已有较大浮盈 → 自动降低新开仓意愿
