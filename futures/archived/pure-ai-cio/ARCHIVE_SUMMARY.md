## 二、策略设计回顾

### 2.1 核心理念
- **第二段行情**：只交易 H1 级别回调确认后的第二段趋势
- **Trade Gate**：5 条硬过滤规则，命中任意 1 条禁止交易该品种
- **净值优先**：回撤控制 > 盈利
- **软约束**：用户坚持用「原则/思维模型」替代硬性规则

### 2.2 交易逻辑
```
D1 确认趋势 → H1 寻找推进段 → 等待回调 ≥ 30% → 回调结束 → 第二段入场
SL = 结构证伪位（突破回调低点/高点）
TP = RR ≥ 1:1（硬性门槛）
动态仓位：单笔风险 1%~5% 净值
```

### 2.3 使用的系统资源
- 2 个 Cron 定时任务（日间 + 周末凌晨）
- pre_analyze.py（实盘数据扫描）
- execute_trade.py（开仓执行）
- fetch_news.py（3 轮深度新闻搜索）
- 交易日志（logs/scans/ + logs/trades/）

## 三、失败原因分析

### 🔴 原因 1：回撤幅度定义不严谨（核心败因）

策略要求回调 ≥ 推进段的 30% 才视为「健康回踩」，但**没有上限**。

**实际案例**（GBPUSD 5/7）：
```
推进段 39.6 pip → 回调 51% ← 超过一半都跌回去了
AI 判断：结构完好 ✅  实盘结果：亏损
```
**本质问题**：回撤超过推进段的 40% = 趋势动能衰竭，不是回调。AI 没有上限约束，导致在趋势逆转处接盘。

### 🔴 原因 2：Trade Gate 5 条规则全是主观判断

5 条规则（D1 震荡、H1 区间、回调未完、假突破、波动率不足）没有一条有量化阈值，全靠 AI 自己判断。

**结果**：AI 可以在任何品种上找理由「通过」Trade Gate，也可以在全部 14 个品种上找理由「拦截」——取决于它想交易还是想偷懒。

### 🔴 原因 3：AI 立场悖论——"不交易"和"积极开仓"自相矛盾

- Phase B 的目标是「积极开仓积累交易数据」
- 但策略 Prompt 的核心口号是「不交易是完全合法且经常是最优决策」
- AI 被夹在中间，开仓则违反「不交易」，不开仓则违反「积极开仓」

### 🔴 原因 4：没有客观的熔断机制

连续 20 笔亏损没有自动停盘机制，全靠 AI「自行判断」。净值从 $2,850 跌到 $1,500 的过程中，没有任何硬性熔断触发。

### 🔴 原因 5：宏观风险层未接入

5/9 非农超预期 + 鹰派 + 布油破 $101 的宏观环境下，策略没有自动避险，仍然在开多单。

## 四、经验教训

### 教训 1：AI 交易策略需要「客观量化」的硬约束
AI 擅长模糊判断，但交易最怕模糊。关键参数必须有数字：
- 「回调幅度 ≥ 30%」→ ✅ OK
- 「回调幅度 ≤ 40%」→ ❌ 缺失
- 「波动率不足」→ ✅ 应改为 ATR/Price < 0.2%
- 所有 Trade Gate 规则都应可量化

### 教训 2：净值优先必须有代码级熔断
```
IF 连续 3 笔亏损 → 当日禁止开仓
IF 净值回撤 > 20% → 停盘 24 小时
IF 回撤 > 30% → 停盘至人工恢复
```
这些必须是 execute_trade.py 中的硬逻辑，不能指望 AI 自觉。

### 教训 3：「软约束」不适合交易策略
用户偏好「原则/思维模型」而非硬性代码规则，但这导致：
- AI 可以自由解释规则
- 同样的规则在不同的扫描中执行效果完全不同
- 无法回测验证策略的历史表现

### 教训 4：模拟盘也要有回撤容忍度上限
Phase B 虽然目标是「积累数据」，但 -47% 的回撤已经失去了统计意义——后面的交易是在恐慌/下行通道中进行的，数据有严重偏差。

### 教训 5：先回测，后实盘
Pure AI CIO 没有经过任何历史回测就上线了。后续策略应该：
1. 量化规则 → 2. 历史回测 → 3. 样本外验证 → 4. 影子模式 → 5. 小资金实盘

## 六、可复用的正确决策

虽然整体失败，但也有一些正确的设计思路值得保留：

1. ✅ **结构证伪止损**：SL 放在「跌破这里趋势判断就错了」的位置，而非固定点数
2. ✅ **动态手数计算**：用净值百分比计算手数，比固定手数科学
3. ✅ **日志完整**：每一笔交易都有详细的决策过程记录
4. ✅ **离场分类标签**：[TP_HIT] [SL_HIT] [STRUCTURE] [TIME_EXIT] [NEWS_RISK]
5. ✅ **不交易也是决策**：理念正确，只是执行上没有量化标准
