# 期货量化交易系统 — 概念说明

## 一句话

这是一个期货和外汇的量化交易系统，从行情接入、策略研究、信号注入到自动下单，全流程自动化运转。

## 数据

实时行情来自 MT5（Exness 模拟盘），通过一个叫 Tick Engine 的组件每秒读取 19 个品种的 tick 数据，同时检测新 K 线形成并计算技术指标，结果写入共享文件。其他模块只读这些文件，不再重复连接 MT5。

品种池覆盖外汇（EURUSD、GBPUSD、USDJPY 等 7 个）、贵金属（XAUUSD、XAGUSD）、指数（US30、US500、USTEC、JP225、HK50）、能源（USOIL、UKOIL）和商品（天然气、铜），一共 19 个。DXY 作为辅助过滤数据，不直接交易。

时间框架覆盖 M1、M5、M15、M30、H1 五个级别。

研究用的历史数据存 Parquet 格式，通过数据更新脚本从 MT5 拉取补齐。指标计算代码和实盘侧是同一份，保证研究结论可以直接落地。

## 指标

系统有一个统一的指标库，包含 84 个函数、320 个技术指标。分几大类：

- 趋势类：各种周期的 EMA、SMA、Ichimoku、Guppy、PSAR
- 动量类：RSI（多周期）、MACD、Stochastic、CCI、Williams %R
- 波动率类：ATR（多周期）、Bollinger Bands、Keltner、Donchian
- 成交量类：OBV、MFI、CMF、Volume Spike
- 价格行为：Doji、Hammer、Engulfing 等 K 线形态识别
- 结构类：HH/LL 突破、支撑阻力、Pivot Points

Tick Engine 每检测到一根新 K 线就会对那根 K 线算全部指标，写入共享 JSON 文件。研究批量预计算也用同一份代码算完后存 Parquet。两边指标值完全一致，不存在研究信号实盘跑不出来的情况。

## 研究

研究通过因子挖掘引擎来做。引擎加载指定数据集，扫描所有指标列，枚举各种阈值组合，评估每条规则的胜率、盈亏比、Sharpe、最大回撤等指标，把合格的发现存入最佳发现池。

目前有四条研究线在并行运转：

- Scalping M1/M5：超短线高胜率策略，找 RSI 极值入场点，胜率要求高
- Intraday M30/H1：日内波段反转策略，结合 RSI 超卖和 DXY 美元方向过滤
- High-RR M5/M15/H1：低胜率高盈亏比策略，找趋势回调或假突破形态，Sharpe 和盈亏比优先
- K 线形态 H1/D1：单根或多根 K 线组合形态的信号挖掘

每条研究线独立维护一个发现池，每次新轮次的结果和已有发现合并排序，淘汰排名靠后的，始终保持一定容量的高质量发现池。

研究由 AI Agent 驱动，定时触发，自动跑引擎、分析结果、写报告。报告和研究状态都存文件，方便事后复盘。

## 注入

研究发现不会直接进交易系统，而是经过一个注入环节。注入脚本读取发现池，按 Sharpe、盈亏比、样本量等门槛过滤，把合格的信号转成标准条件格式，追加到策略配置文件里。

注入过程是全自动的，不需要手写代码。每个注入的信号都带有来源标记，不会重复注入。

条件格式支持三种模式：简单字段（比如设置 RSI 和 session 阈值）、通用条件数组（可以表达任意指标的组合条件）、以及特殊处理器（用于跨品种 DXY 过滤、跨时间框架 H1 趋势判断、或检查价格是否在均线附近）。这三种格式在底层统一求值，扫描器不需要区分信号是哪种格式来的。

## 交易

实盘执行层由多个 Autopilot 组成，每个独立运行在 tmux 会话里，通过 systemd 管理生命周期。

每个 Autopilot 做的事情很简单：死循环里读策略配置、读共享行情数据、遍历所有信号检查条件是否触发、过风控、下单。

目前有三个 Autopilot 在运行：
- Scalping AP：专注 M1/M5 超短线，Magic 234011
- Intraday AP：专注 M30/H1 波段，Magic 234010
- Scalping PA：专注价格行为信号，Magic 234013

每个 Autopilot 有不同的风险参数：持仓数上限、同品种不重复、同组不重复、最小盈亏比。所有开仓必须带止损止盈，止损参考 ATR 计算，止损倍数和止盈倍数可以在信号配置里单独指定。

日志方面，每次扫描结果、每次交易执行都存 JSON 文件按时间戳命名，触发器文件供 AI 复盘读取。

## 部署

系统组件全部跑在 Windows + WSL 混合环境。Tick Engine 和 Autopilot 跑在 Windows Python 下（需要 MT5 的 Python API），AI Agent 和研究脚本跑在 WSL 下。

所有组件通过 tmux 管理，外面再套一层 systemd 保证开机自启。每个组件被一个 bash while 循环包裹，崩溃后等待若干秒自动重启。

新增策略只需要在 daemon 注册表里加一条 JSON 记录，指定脚本路径和重启延迟，不需要改启动脚本或 systemd 配置。

系统 crontab 里还有一条每分钟触发的扫描任务，作为 Autopilot 的补充，确保即使主循环出问题也有兜底扫描。

## 信号现状

当前 Intraday 引擎配置了 74 个信号（其中 12 个是原始研究产出的，47 个从 Scalping 研究成果注入的，15 个从 High-RR 研究成果注入的），Scalping 引擎配置了 54 个信号。合计 128 个交易信号在自动扫描中。

信号按优先级排序，优先级高的信号先执行。信号覆盖了 19 个品种、5 个时间框架、多空两个方向。实际是否触发取决于当时的市场条件是否满足信号的条件组合。
